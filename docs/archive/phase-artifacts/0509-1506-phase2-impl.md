# Phase 2 polish: 配置 modal 嵌套 fieldset 溢出修复

## 范围
仅修 `frontend/styles.css`，不动 HTML / JS / backend。

## 用户报告
配置 modal → 「LLM 智能摘要」 fieldset → 内嵌的 `provider-block[data-provider=anthropic|openai]` fieldset：
- label 长说明（含 latin token / 括号内长串 / URL 候选列表）横向溢出 fieldset 边线
- input placeholder 长 URL 列表也存在容器撑爆风险
- 嵌套 fieldset 视觉层级不清

## ui-ux-pro-max 决策（form polish 范畴，沿用 Phase 2 既有 design system）
查询：`form input nested fieldset overflow long label` --domain ux

| Result | 取舍 |
|---|---|
| R1/R7 label 必须可见 | 不裁切 label 文本，改为换行 |
| R2 慎用 overflow-hidden | label 用 `overflow-wrap: anywhere` 自然换行而非 hidden 切掉 |
| R4 长内容优雅截断 | input 加 `text-overflow: ellipsis` 兜底（placeholder 默认即 clip） |
| R8 行长 65-75ch | modal 460px 宽天然限制行长，无需额外控制 |

决策：
- pattern: 嵌套 fieldset 用 `subtle bg + hairline border + 缩进 padding` 形成层级，不加深嵌套外框
- style: 沿用 Phase 2 minimalism + Swiss 2.0（不引入新风格）
- color: 全部走 token（`--c-subtle` / `--c-hairline` / `--c-fg-soft`）
- typography: 内层 legend `--fs-xs/600`、不 uppercase 不 letter-spacing（中文 + 短词不适合）；外层 legend 保持原 uppercase
- motion: 不加
- stack: vanilla CSS（无变更）

## 修改清单（styles.css，5 处）

| selector | 改动 | 目的 |
|---|---|---|
| `.modal-card` | 加 `overflow-x: hidden` | 兜底防横向滚动条出现 |
| `.modal-card label` | 加 `overflow-wrap: anywhere` / `word-break: break-word` / `min-width: 0` / `line-height: var(--lh-base)` | 让 label 长 latin token / URL / 括号说明在任意位置自然换行 |
| `.modal-card label.row > input[type="checkbox"]` | 新增 `flex-shrink: 0` | 防 row label 在 flex 收缩时挤压复选框 |
| `.modal-card input[type=text|password|number]` | 加 `min-width: 0` / `max-width: 100%` / `text-overflow: ellipsis` | 防御性容错 + 长 value 截断 |
| `.modal-card select` | 新增 `max-width: 100%` | 防 select 长 option 撑爆 |
| `.modal-card fieldset` | 加 `min-width: 0` | flex/grid 上下文里防内部宽元素溢出 |
| `.modal-card fieldset fieldset` | **新增** `background: var(--c-subtle)` / `border: var(--bd-hair)` / `padding: var(--sp-2) var(--sp-3) var(--sp-1)` / `margin: var(--sp-2) 0 var(--sp-3)` | 嵌套层视觉层级：浅底 + 发丝边 + 紧凑内距，比外层弱 |
| `.modal-card fieldset fieldset legend` | **新增** `--fs-xs / 600` / `color: --c-fg-soft` / `text-transform:none` / `letter-spacing:0` | 内层 legend 字重做层级，不沿用 uppercase（中文 + 短词不合适） |
| `.modal-card fieldset fieldset label` | **新增** `margin-bottom: var(--sp-2)` / `color: var(--c-fg)` | 内层 label 间距更紧、文本对比更高 |

## 关键决策
1. **label 换行而非截断**：长说明文字（含 endpoint URL 列表）属于"决策性提示"，不可被 ellipsis 隐藏；用 `overflow-wrap: anywhere` 在任意字符断行，配合 `--lh-base: 1.55` 多行可读。
2. **嵌套层 subtle bg 而非粗描边**：spec §2 原则三"边比框轻 / 间距比边重" → 用浅底 + hairline 表达层级，比加粗外框更克制。
3. **内层 legend 去 uppercase**：中文 "Anthropic 配置 / OpenAI 兼容配置" + uppercase 在 CJK 字体里没有视觉效果，且和外层 legend "推送策略 / 推送内容 / LLM 智能摘要" 区分不开 → 改用字重 (600 vs 外层 500) 拉层级。
4. **box-sizing 已全局 border-box**（styles.css:62），不再重复。
5. **不动 HTML 文案**：用户原话是"字体超过模块栏" = CSS 问题，文案本身（"模型（按你选的 endpoint 改：...）"等）是产品决定，不修。

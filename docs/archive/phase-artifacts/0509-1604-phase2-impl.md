# Phase 2 polish — LLM fieldset .muted.small（2026-05-09 16:04）

## 上下文
上一轮（0509-1506）已统一 modal 嵌套 fieldset 溢出与层级；本轮用户在 HTML 端给每个 provider-block fieldset 顶部新增 `<p class="muted small">` 一句话说明，CSS 还未对这个新组合做规则。

## ui-ux-pro-max 校准
- query: `form fieldset legend helper text spacing` --domain ux
- 命中：Result 6 `color-contrast`（4.5:1）/ Result 7 `contrast-readability`（gray on gray 禁忌）
- 决策：fieldset 内 helper text 必须从 `--c-fg-muted`（#8a8a82）升级到 `--c-fg-soft`（#5a5a55），在嵌套层 `--c-subtle`（#f0efec）浅底上达 WCAG AA

## 改动（仅 frontend/styles.css，已有 modal 段尾部）

新增 3 条 selector：

1. `.modal-card fieldset > .muted.small, .modal-card fieldset > p.muted.small`
   - `margin: 0 0 var(--sp-2)`：reset `<p>` 浏览器默认 margin-block；与下方第一个 label 留统一 `--sp-2` 节奏
   - `color: var(--c-fg-soft)`：浅底上对比度达标；同时与上层 legend (`--c-fg-muted` 大写) 形成"标签 vs 说明"层级对比
   - `line-height: var(--lh-tight)` (1.35)：一句话宁紧不松，避免 12px 文本撑两行时的视觉膨胀

2. `.modal-card label > select { width: 100% }`
   - select 撑满 label 内宽，与同级 input 对齐
   - 长 option（`--bare（需 ANTHROPIC_API_KEY）`）关闭态由浏览器原生 ellipsis 截断；展开层是浏览器原生绘制，不在 CSS 控制范围内（不强求换行）

## 设计选择对照

| 维度 | 决策 | why |
|---|---|---|
| 对比度 | `--c-fg-soft`（不是 `--c-fg-muted`） | WCAG 4.5:1；ux skill Result 6/7 直接命中 |
| 顶部 margin | 0 | 紧贴 legend 不留视觉缺口（legend 已上移到边线） |
| 底部 margin | `--sp-2` | 与嵌套 label 的 `margin-bottom: var(--sp-2)` 同节奏 |
| 字号 | 12px（沿用 .small） | 已是辅助层级，不再降；满足 readable-font-size |
| 行高 | `--lh-tight` (1.35) | 一句话辅助文本，紧凑 |

## 不动
- 不动 HTML 文字内容
- 不动 token / 不动颜色变量定义
- 不动 JS / aliases modal / 顶层 LLM fieldset
- 不动上一轮已稳定的嵌套层级规则（`--c-subtle` + `--bd-hair`）

## 风险
- `<p class="muted small">` 全局其它位置若也用作"非 fieldset 内 helper"，本规则不影响（用了 `> ` direct child 选择器约束 scope）
- form 末尾 `<p id="config-msg" class="muted small">` 是 form 直接子节点，**不在** fieldset 内，本规则不命中（不会被覆盖，保持原 `--c-fg-muted`）—— 已验证选择器 scope 正确
